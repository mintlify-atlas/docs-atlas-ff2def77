## Overview

Categories help you organize and classify your financial movements. WalletSync supports two types of categories: `EXPENSE` for outgoing payments and `INCOME` for incoming money. You can also create word associations to automatically categorize transactions.

## Category Endpoints

All category operations are available under:

```
/api/categories
```

## Category Types

Categories must be one of two types:

| Type | Description | Example |
|------|-------------|----------|
| `EXPENSE` | Outgoing payments | Groceries, Rent, Entertainment |
| `INCOME` | Incoming money | Salary, Freelance, Dividends |

## Creating Categories

### Create a New Category

**Endpoint:** `POST /api/categories`

**Request Body:**
```json
{
  "categoryName": "Groceries",
  "categoryType": "EXPENSE"
}
```

**Response (201 Created):**
```json
{
  "id": 1,
  "categoryName": "Groceries",
  "categoryType": "EXPENSE"
}
```

### API Examples

<CodeGroup>
```bash cURL
curl -X POST http://localhost:8080/api/categories \
  -H "Content-Type: application/json" \
  -d '{
    "categoryName": "Groceries",
    "categoryType": "EXPENSE"
  }'
```

```bash HTTPie
http POST http://localhost:8080/api/categories \
  categoryName="Groceries" \
  categoryType="EXPENSE"
```

```javascript JavaScript (Fetch)

    categoryName: 'Groceries',
    categoryType: 'EXPENSE'

```
</CodeGroup>

## Deleting Categories

### Delete a Category

**Endpoint:** `DELETE /api/categories/{id}`

**Response (204 No Content):** Category deleted successfully

**Response (500 Internal Server Error):** Deletion failed

**API Examples:**

<CodeGroup>
```bash cURL
curl -X DELETE http://localhost:8080/api/categories/1
```

```bash HTTPie
http DELETE http://localhost:8080/api/categories/1
```

```javascript JavaScript
fetch('http://localhost:8080/api/categories/1', {
  method: 'DELETE'
})
.then(response => {
  if (response.status === 204) {
    console.log('Category deleted successfully');
  }
});
```
</CodeGroup>

<Warning>
  Deleting a category may affect associated transactions and word associations. Ensure the category is no longer needed before deletion.
</Warning>

## Category Associations

Word associations allow automatic categorization of transactions based on description patterns.

### Association Endpoints

All association operations are available under:

```
/api/associations
```

## Creating Associations

### Create Word Association

**Endpoint:** `POST /api/associations`

**Request Body:**
```json
{
  "associationWord": "Netflix",
  "categoryId": 3
}
```

**Response (201 Created):**
```json
{
  "id": 1,
  "associationWord": "Netflix",
  "categoryId": 3,
  "categoryName": "Entertainment"
}
```

<Note>
  The response includes `categoryName` for convenience, but you only need to provide `associationWord` and `categoryId` when creating.
</Note>

## Controller Implementation

### Category Controller

Here's the implementation of the category management endpoints:

```java
@RestController
@RequestMapping("/api/categories")
@RequiredArgsConstructor
public class CategoryController {

    private final CategoryService categoryService;

    @GetMapping
    public ResponseEntity<List<CategoryDTO>> getAll() {
        return ResponseEntity.ok(categoryService.findAll());
    }

    @GetMapping("/{id}")
    public ResponseEntity<CategoryDTO> getById(@PathVariable Long id) {
        return ResponseEntity.ok(categoryService.findById(id));
    }

    @PostMapping
    public ResponseEntity<CategoryDTO> create(@RequestBody CategoryDTO categoryDTO) {
        CategoryDTO savedCategory = categoryService.save(categoryDTO);
        return new ResponseEntity<>(savedCategory, HttpStatus.CREATED);
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        boolean response = categoryService.delete(id);
        if(response) {
            return new ResponseEntity<>(HttpStatus.NO_CONTENT);
        } else {
            return new ResponseEntity<>(HttpStatus.INTERNAL_SERVER_ERROR);
        }
    }
}
```

### Association Controller

```java
@RestController
@RequestMapping("/api/associations")
@RequiredArgsConstructor
public class MovementCategoryAssociationController {

    private final MovementCategoryAssociationService associationService;

    @GetMapping
    public ResponseEntity<List<MovementAssociationDTO>> getAll() {
        return ResponseEntity.ok(associationService.findAll());
    }

    @PostMapping
    public ResponseEntity<MovementAssociationDTO> create(
        @RequestBody MovementAssociationDTO dto) {
        MovementAssociationDTO saved = associationService.save(dto);
        return new ResponseEntity<>(saved, HttpStatus.CREATED);
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        associationService.delete(id);
        return ResponseEntity.noContent().build();
    }
}
```
